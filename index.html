Django

Django简介

Django是python的web开发框架，是目前python web框架中功能最全面，文档最完整的一个。Django的理念是DRY(Don’t Repeat Yourself)来鼓励快速开发！目前想自己着手做一个ios app，身边没有朋友能写服务器端程序，自己决定快速学习一下python和django这套框架。
框架结构
urls.py
网址入口，关联到对应的views.py中的一个函数（或者generic类），访问网址就对应一个函数。
views.py
处理用户发出的请求，从urls.py中对应过来, 通过渲染templates中的网页可以将显示内容，比如登陆后的用户名，用户请求的数据，输出到网页。
models.py
与数据库操作相关，存入或读取数据时用到这个，当然用不到数据库的时候 你可以不使用。
forms.py
表单，用户在浏览器上输入数据提交，对数据的验证工作以及输入框的生成等工作，当然你也可以不使用。
templates 文件夹
views.py 中的函数渲染templates中的Html模板，得到动态内容的网页，当然可以用缓存来提高速度。
admin.py
后台，可以用很少量的代码就拥有一个强大的后台。
settings.py
Django 的设置，配置文件，比如 DEBUG 的开关，静态文件的位置等。

Django的基本命令

django-admin.py startproject project-name //新建项目
python manage.py startapp app-name 
django-admin.py startapp app-name//新建app，上也是

python manage.py syncdb //同步数据库
注意：Django 1.7.1及以上的版本需要用以下命令
python manage.py makemigrations
python manage.py migrate

终端上输入 python manage.py 可以看到详细的列表，在忘记了名称的时候特别有用。

GitHub和Hexo搭建博客

一.准备工作

1.安装node.js

(安装brew：)

$ ruby -e "$(curl -fsSL https://raw.github.com/Homebrew/homebrew/go/install)”

安装node.js

$ brew install node

2.安装Hexo

$ npm install -g hexo
$ hexo init myBlog
$ npm install
我在执行$ npm install的时候出现了这个错误：
npm ERR! install Couldn't read dependencies
npm ERR! Darwin 14.1.0
npm ERR! argv "node" "/usr/local/bin/npm" "install"
npm ERR! node v0.12.4
npm ERR! npm  v2.10.1
npm ERR! path /Users/frank/package.json
npm ERR! code ENOPACKAGEJSON
npm ERR! errno -2
npm ERR! package.json ENOENT, open '/Users/frank/<p>package.json'
npm ERR! package.json This is most likely not a problem with npm itself.
npm ERR! package.json npm can't find a package.json file in your current directory.
解决办法：$ sudo npm install -g hexo-cli
执行$ hexo遇到如下提示：

解决：重新回到myBlog目录下执行$ npm install

二.配置

$ cd myblog
$ vim  _config.yml
结尾处做如下修改：
# Deployment
#Docs: http://hexo.io/docs/deployment.html
deploy:
 type: git
repository: git@github.com:frankever/frankever.github.io.git
branch: master
你可能会遇到这样的错误，提示
ERROR Deployer not found: github
或者：
ERROR Deployer not found: git
以前版本的是github，现在需要改成git，并执行npm install hexo-deployer-git --save
还有个问题需要注意：配置文件里面＃和:后面是有个空格的，如果没有会提示
JS-YAML: can not read a block mapping entry; a multiline key may not be an implicit key at line 13, column 1:
# URL
^
配置文件里面还有一些需要我们暂时了解一下：
# Site
title: 博客标题
subtitle: 副标题
description: 描述
author: 作者
language: 语言
timezone: Asia/Shanghai时区

#URL（自己域名的url）
 If your site is put in a subdirectory, set url as 'http://yoursite.com/child'     and root as '/child/'
url: http://www.lolbella.com
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:
如果以上修改都做好了，我们不妨预览一下：

$ hexo g


$ hexo s

打开http://localhost:4000看看有没有出现博客的页面.

三.发布到github上

1.首先去注册一个github账户，然后创建一个仓库new repository.比如说我的账户是frankever，我创建一个名为frankever.github.io的仓库，大家改一下username就好了，别的都是一样的.

$ hexo d

将博客项目上穿到仓库里面
